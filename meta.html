Semantic in Detail

  • Introduction

    Semantic HTML is the use of HTML markup to reinforce the semantics, or meaning, of the information in webpages and web applications rather than merely to define its presentation or look.

    Semantic HTML is processed by traditional web browsers as well as by many other user agents.

  • What are Semantic Tags?

    A semantic element clearly describes its meaning to both the browser and the developer.

    Search engines will consider its contents as important keywords to influence the page's search rankings.

    Screen readers can use it as a signpost to help visually impaired users navigate a page.

    Finding blocks of meaningful code is significantly easier than searching through endless divs with or without semantic or namespaced classes.

    Suggests to the developer the type of data that will be populated.

  • Why should you use Semantic Tags?

    The benefit of writing semantic HTML stems from what should be the driving goal of any web page: the desire to communicate. By adding semantic tags to your document, you provide additional information about that document, which aids in communication.

    Specifically, semantic tags make it clear to the browser what the meaning of a page and its content is. That clarity is also communicated with search engines, ensuring that the right pages are delivered for the right queries.

  • Semantic Tag Best Practices

    When using semantic tags to convey meaning rather than for presentation purposes, be careful that you don't use them incorrectly simply for their common display properties. Some of the most commonly misused semantic tags include:

    • Blockquote Tags

      blockquote — Some people use the blockquote tag for indenting text that is not a quotation. This is because blockquotes are indented by default. If you simply want to indent text that's not a blockquote, use CSS margins instead.

    • Paragraph Tags

      Some web editors use a non-breaking space contained in a paragraph to add extra space between page elements, rather than defining actual paragraphs for the text of that page. we should use the margin or padding style property instead to add space.​

    • Unordered List Tags

      As with blockquote, enclosing text inside a unordered list tag indents that text in most browsers. This is both semantically incorrect and invalid HTML, because only list tags are valid within a unordered list tag. Again, use the margin or padding style to indent text.

    • h1, h2, h3, h4, h5, and h6

      You can use heading tags to make fonts bigger and bolder, but if the text is not a heading, use the font-weight and font-size CSS properties instead.

    • Sections and Article

      The section tag defines sections in a document, such as chapters, headers, footers, or any other sections of the document.

      The article tag specifies independent, self-contained content. An article should make sense on its own and it should be possible to distribute it independently from the rest of the site.

  • Conclusion

    An important principle in web design is the idea of using HTML elements to indicate what they actually are, rather than how they may appear in the browser by default. This is known as using semantic HTML. By using HTML tags that have meaning, you create pages that impart more information than those that simply surround everything with div tags
